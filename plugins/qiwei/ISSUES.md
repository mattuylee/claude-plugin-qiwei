# 待解决问题

## 已知限制

1. **企微文件消息类型不确定**：部分企微版本中文件消息可能通过 event 类型推送而非 MsgType=file，需要在实际测试中确认并适配。

2. **语音消息识别**：语音消息的 Recognition 字段需要在企微后台开启语音识别功能才会有值，否则只能拿到 MediaId。

3. **消息长度限制**：企微文本消息发送 API 限制单条消息最大 2048 字节，超长消息需要分段发送，当前实现未做分段处理。

4. **临时素材有效期**：企微临时素材有效期为 3 天，过期后 media_id 无法下载。如果用户较晚调用 download_attachment 可能会失败。

5. **并发消息处理**：当前 callback server 对消息的处理是串行的（异步但逐条），高并发场景下可能需要队列化。

6. **access_token 并发刷新**：多个请求同时发现 token 过期时，可能触发多次刷新。企微 API 对 gettoken 调用频率有限制（每小时不超过若干次），当前未加锁。

## 待确认事项

1. 企微自建应用回调 URL 的具体路径格式——当前同时监听 `/` 和 `/callback`，需确认企微后台配置时填入的 URL 路径。

2. 是否需要支持企微的事件推送（如关注/取消关注、菜单点击等），当前仅处理消息推送。
